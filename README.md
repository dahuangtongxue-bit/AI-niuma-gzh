# 数字员工 · 阿文｜公众号运营专员（MVP v1）

每天交付 **1 篇头条级成稿**：排版正文（复制即贴进公众号后台，所见即所得）+ 标题 Top3（10 进 3 打分淘汰）+ 头图（900×383）+ 金句卡 2 张（1080×1440）+ 群发摘要 + 发布建议。

公众号是转发逻辑——这位员工的一切产出围绕"值得转发"设计：选题看转发价值、金句专为朋友圈裂变、结尾克制引导在看。与小红书（点击逻辑）、抖音（完播逻辑）是三套不同的生产工艺。

---

## 一、部署（Netlify，3 步）

1. 推 GitHub 仓库（或拖 zip 到 Netlify）
2. New site → 选仓库，构建命令 `npm run build`（netlify.toml 已配好）
3. 环境变量（与阿桃/阿抖同一套 schema，可共用零克云网关）：

| 变量 | 必填 | 说明 |
|---|---|---|
| `LLM_API_BASE` | ✅ | OpenAI 兼容网关，到 `/v1` 为止 |
| `LLM_API_KEY` | ✅ | 网关 Key |
| `LLM_MODEL` | ✅ | 主笔模型（长文质量取决于它，建议用写作强的） |
| `LLM_MODEL_JUDGE` | 可选 | 标题评委模型，便宜快的即可；空则同 LLM_MODEL |

本地：`cp .env.example .env.local` → `npm install` → `npm run dev`

生产流程：选题海选（5 个带推荐指数，定稿 1 个，落选连理由进日志）→ 大纲+10 标题（结构先行）→ **成文与标题评分并行** → 标题 10 进 3（评委 4 维+代码硬规则合并）→ 质检机检 → 硬规则不达标自动打回返工 1 次复检 → 排版引擎出内联样式稿 → 头图+金句卡 → 交付。

## 三、产品设计要点（"高标准"落在哪）

- **排版直出**：`wxFormat.js` 把文章 JSON 渲染成全内联样式 HTML（公众号编辑器剥 class 留 inline style，秀米同理），「复制带排版正文」走 ClipboardItem 富文本通道（降级 execCommand），贴进后台所见即所得——序号徽章小标题、主题色金句块、重点标记、文末在看引导块全部就位
- **结构先行**：大纲与成文分两阶段，长文质量的生命线；开头按钩子库 12 式落笔，首段 ≤100 字机检
- **金句是转发发动机**：每节配金句，全文提炼 2 句最强的制成署名金句卡，读者转卡=免费投放
- **平台规则差异内化**：诱导分享（不转不是/集赞/转发抽奖）为微信红线 block 级；引导加微信/进群属私域常规，不封禁——与小红书规则相反，写在 `bannedWords.js` 注释里
- **手机框预览**：375px 宽手机框直接渲染排版稿，验收不用想象

## 四、v1 边界

- 头图/金句卡为模板排版（纯文字+主题色），图片底版式待接生图
- 热点半自动投喂；新榜/微信指数等数据源 v2 接
- 单用户 localStorage；手动开工；次条/多图文排期待 v1.1

## 五、路线图

v1.1 次条内容（清单/问答轻量稿）+ 多图文组合 → v1.2 数据复盘闭环（阅读/在看率投喂 → 调选题情绪浓度与金句密度）→ v1.3 生图头图底 → 与阿桃/阿抖合并「数字员工花名册」统一入口
